HTML Page Overview

Notes from the video explaining file paths, boilerplate, semantic tags, and HTML basics.

I. File Path System

The file path is like a set of directions for finding a file.

  • Relative Path: Changes depending on the folder location. Uses symbols like ./ (same folder) and ../ (go up one folder).
  • Absolute Path: The full path from the root or a complete URL. Does not change with location.
  • Use Cases: Relative paths are best for project files. Absolute paths are useful for external resources.

II. HTML Boilerplate

Basic structure of every HTML page.

  • !DOCTYPE html - Defines document type as HTML5.
  • html lang="en" - Root element with language set.
  • head - Metadata, title, and settings.
  • title - Page title in the browser tab.
  • meta charset="UTF-8" - Supports all characters.
  • meta name="viewport" - Makes pages responsive on mobile devices.
  • body - The visible content of the page.

III. Semantic Elements

Elements that describe meaning and structure of content.

  • header - Top section, usually logo or navigation.
  • main - Core unique content of the page.
  • footer - Bottom part with copyright or links.
  • Benefit: Improves SEO and accessibility.

IV. Core HTML Concepts

  • div - A container to group content.
  • class - Used to label multiple elements in the same way. Example: div class="container".
  • id - A unique identifier for one element. Example: h1 id="title".

V. Linking External Files

  • link rel="stylesheet" href="style.css" - Adds external CSS.
  • script src="app.js" script - Adds external JavaScript.
  • Best practice: Organize files into folders like css, js, and images.

VI. File and Naming Best Practices

  • Use lowercase file names (example: index.html).
  • Use hyphens instead of spaces (example: about-us.html).
  • Keep related assets organized in folders.
